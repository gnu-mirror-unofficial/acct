Introduction to the GNU Accounting Utils

The GNU Accounting Utils provide login and process accounting utilities for GNU/Linux systems.

Downloading the GNU Accounting Utils

The latest release of the GNU Accounting Utils can be found in the subdirectory /gnu/acct/ on your favorite GNU mirror. For other ways to obtain the GNU Accounting Utils, please read How to get GNU Software

Documentation

A GNU Info manual is included in the GNU Accouting Utils package.

You may also find more information about the GNU Accounting Utils by looking at /usr/doc/acct/ or /usr/local/doc/acct/ or by looking at man pages (man acct at the shell prompt) on your system.

Mailing Lists/Newsgroups

The GNU Accounting Utils now has two mailing lists: <bug-acct@gnu.org>, for bug reports, and <help-acct@gnu.org> for questions about using or development work with the GNU Acct.

To subscribe to any the GNU Accounting Utils mailing list, please send an empty mail with a Subject: header line of just "subscribe" to the relevant -request list. For example, to subscribe yourself to the main list, you would send mail to <bug-acct-request@gnu.org> with no body and a Subject: header line of just "subscribe".

Request an Enhancement

If you would like any new feature to be included in future versions of the GNU Accounting Utils, please send a request to <bug-acct@gnu.org>

Please remember that development of the GNU Accounting Utils is a volunteer effort, and you can also contribute to its development. For information about contributing to the GNU Project, please read How to help GNU.

Report a Bug

If you think you have found a bug in the GNU Accounting Utils, then you should send as complete a report as possible to <bug-acct@gnu.org>. If you would like to submit a patch, please use diff -u5 to generate the patch, and give a detailed description.

Development

The GNU Accounting Utils is currently being maintained by Ciaran O'Riordan <ciaran@member.fsf.org>. The project is managed online at Savannah. Current development is done by Ciaran O'Riordan and Manuel A. Fernandez Montecelo.
