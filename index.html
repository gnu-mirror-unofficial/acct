Introduction to the GNU Accounting Utils

This is a web page about the GNU Accounting Utils.

Downloading the GNU Accounting Utils

the GNU Accounting Utils can be found on in the subdirectory /gnu/acct/ on your favorite GNU mirror. For other ways to obtain the GNU Accounting Utils, please read How to get GNU Software

Documentation

the GNU Accounting Utils documentation can be found at http://www.gnu.org/manual/manual.html

You may also find more information about the GNU Accounting Utils by looking at /usr/doc/acct/ or /usr/local/doc/acct/ or by looking at man pages (man acct at the shell prompt) on your system.

Mailing Lists/Newsgroups

the GNU Accounting Utils now has two mailing lists: <help-acct@gnu.org> and <bug-acct@gnu.org>.

The main discussion list is <help-acct@gnu.org>, and is used to discuss all aspects of the GNU Accounting Utils, including development and porting.

Announcements about the GNU Accounting Utils and most other GNU Software are made on <info-gnu@gnu.org>.

There is also a special list used for reporting bugs, <bug-acct@gnu.org>. For details on submitting a bug report, please see the section Report a Bug below.

To subscribe to any the GNU Accounting Utils mailing list, please send an empty mail with a Subject: header line of just "subscribe" to the relevant -request list. For example, to subscribe yourself to the main list, you would send mail to <help-acct-request@gnu.org> with no body and a Subject: header line of just "subscribe".

Currently, there is one newsgroup dedicated to the GNU Accounting Utils - gnu.acct

Request an Enhancement

If you would like any new feature to be included in future versions of the GNU Accounting Utils, please send a request to <help-acct@gnu.org>

Please remember that development of the GNU Accounting Utils is a volunteer effort, and you can also contribute to its development. For information about contributing to the GNU Project, please read How to help GNU.

Report a Bug

If you think you have found a bug in the GNU Accounting Utils, then you should send as complete a report as possible to <bug-acct@gnu.org>. Ideally, you should include the text you get by running config.guess, the text you see when you run , and configure, and if you can, a patch made with diff -u5 which fixes the problem.

Maintainer
